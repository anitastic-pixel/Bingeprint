Settings
where Smart Picks shows up
Side panel
Adds a sparkle button to Crunchyroll's top bar. Click it to open the ranked Smart Picks list.
Series-page card
Drops a Smart Score card on each series page so you can see why we'd (or wouldn't) recommend it before you click Watch.
Keep tour button on top bar
The tour button (★) auto-hides once you've watched the onboarding tour. Turn this ON to pin it in Crunchyroll's top bar so you can re-fire the tour any time.
Show spoiler tags
Reveal AniList-flagged spoiler tags directly in the rationale chips. Off by default — spoiler-tagged contributors render as a 🔒 placeholder you can tap to reveal per-chip. Turn on if you've already seen most of your catalog.
Dev: axis sandbox
Adds a slider sandbox to the side panel's Shape view. Tweak axis values + signal count to preview how the radar / family palette / shape name / prose render under any combination — no need to watch shows or re-rate. Sandbox state is ephemeral to dev mode and doesn't touch your real taste profile.
Hard excludes — any show with one of these tags as a central theme won't appear in your recs. Tap a suggested tag to treat it as a dealbreaker; tap an active one to remove it.

Show me around again
Re-fires the 5-slide onboarding tour over Crunchyroll (the one you saw on install). Also re-arms one-time coach marks (Picks tab, Shape tab, Smart Score card) so they fire again the next time you open each surface.
Dev: keep onboarding visible
Re-shows the survey "👋 New here?" banner and one-time coach marks every load instead of remembering they were dismissed. Doesn't touch your stored dismissal flags — toggling off restores normal behavior. Used for QAing the new-user experience without resetting storage.
Dev: CF re-ranker
Enables the collaborative-filtering re-ranker on rec list order. Silent by default — reorders Smart Picks only; never changes the displayed score. See docs/CF-RERANKER-DESIGN.md for the design and docs/adr/0003-cf-reranker-ml-exception.md for the no-ML principle override that authorizes it.
Dev: CF Δ pills (side panel)
Renders a per-card "CF Δ" indicator on the side panel showing each rec's CF-engine contribution to the internal ranking score. Audit-only; doesn't affect rec order.
validation (dev)
Generate snapshot
Dump current engine state — top 30 / bottom 30 / per-archetype top-5 / curated case results — to a JSON file. Used to diff against a committed baseline before merging tuning changes. See validation/README.md in the repo for workflow.
Rebuild rec-pool from CR catalog
Enumerate Crunchyroll's full series catalog (~1,950) and resolve new shows to AniList, then download the assembled rec-pool-by-cr-id.json to commit. Long run (~15 min) — watch the engine-status progress.
